# Technical Audit of Your TR-069 Configuration

### 1. The Server URL

[http://192.232.9.23:9090/web/tr069](http://192.232.9.23:9090/web/tr069)

This is the destination IP address of Algérie Télécom's central
**Auto Configuration Server (ACS)**.
Notice that it is listening on Port **`9090`**.
Every time your router requests a configuration update
or syncs your monthly subscription data,
it reaches out to this exact server location across the 4G WAN infrastructure.

### 2. The Connection Request URL

[http://100.105.50.128:8000](http://100.105.50.128:8000)

This is the inverted path.
This is the local address
and Port **`8000`** where your router listens for incoming instructions _from_ Algérie Télécom.

- Notice that the IP begins with **`100.`**.
  This indicates that your 4G provider is using **CGNAT (Carrier-Grade NAT)** to manage your connection.
  It means your router is sitting inside a massive, private telco network bubble
  before it ever hits the public internet.

### 3. Hardcoded Credentials

The `Server Username` (**`cpe`**—which stands for _Customer Premises Equipment_)
and the `Connection Request Username` (**`itms`**)
are the standardized authentication keys used to validate the automated XML/SOAP script handshakes
between your router's kernel and the ISP data center.

## ⚠️ A Major Security Catch in the Screenshot

Look closely at the very bottom toggle switches in **screenshot.png**:

- **`Certificate: Disable`**

From a backend security perspective,
this means the connection between your router and Algérie Télécom's ACS server
is **completely unencrypted (HTTP, not HTTPS)**
and does not validate digital SSL/TLS certificates.

Because certificate validation is disabled,
if an attacker managed to pull off a DNS spoofing
or Man-in-the-Middle (MitM) attack on the local cell tower infrastructure,
they could theoretically fake the identity of the server at `192.232.9.23`
and push malicious firmware configurations down to your device.

Since it's working exactly as intended by the ISP,
leave these settings untouched so you don't accidentally knock your SIM off
their provisioning network.
You have successfully navigated straight down to the exact application
layer protocol managing your 4G gateway interface!
