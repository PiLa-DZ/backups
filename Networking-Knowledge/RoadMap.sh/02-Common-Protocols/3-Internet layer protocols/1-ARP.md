# ARP `Address Resolution Protocol`

ARP maps IP addresses to physical MAC addresses of devices
and vice versa within a LAN so devices can communicate with one another.
ARP is necessary because IP and MAC addresses are different lengths
and operate on different layers of the OSI model.

These addresses must be mapped for proper network communication
and data transfer among connected devices.
ARP isn't required every time devices attempt to communicate
because the LAN's host system maps and stores the associations in its ARP cache.
As a result,
the ARP resolution process is mainly used when new devices join the network.
